Введение

Целью фреймворка является удобная организация работы с данными в JSON-формате.
Таким образом, все это представляет из себя часть паттерна MVC или MVVM, которая имеет отношение к модели, т.е данным.
Благодяря реализации паттерна «наблюдатель» и трэкинга внутренних зависимостей модели (dependent observable), становится возможным гибко управлять логикой приложения и его отображением.
Библиотека не предоставляет никаких возможностей работы с DOM. Для этого используйте jQuery или любой другой фреймворк.

Основные возможности

  • JSON-формат модели
  • Простой и интуитивно понятный API для работы с моделью (syntax sugar)
  • Отслеживание изменений любого объекта, массива или свойства модели (model.follow)
  • Возможность создания составных свойств модели, зависящих от других данных модели (model.composite)
  • Легкая сериализация всей модели или любой её части (model.toJSON)
  • Сохранение и восстановление промежуточных состояний модели (model.backup, model.restore)
  • Обновление модели с автоматическим вызовом всех зависимых колбэков (model.merge)
  • Ручной вызов одного или всех слушателей данных модели (model.dispatch)

Пример кода

var model = Follow('sample'); model({ lang: { from: 'en', to: 'ru' } }); model.follow('lang.to', function( value, params ){ $('#flag').attr('src', '/icons/'+ value + '.png'); }); $('select[name="lang.to"]').change(function(){ model('lang.to', this.value); });
